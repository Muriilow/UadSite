Installation Guide

Important Prerequisites:

  • Read the FAQ before proceeding.
  • Backup your data - You can never be too careful!
  • Remove OEM accounts from your Android device's Settings app before debloating.

Step 1: Install ADB

ADB (Android Debug Bridge) is required for UAD-ng to communicate with your Android device. Choose your operating system:

Windows

Method 1: Using Winget (Recommended)

  1. Install WinGet if not already available
  2. Open PowerShell as Administrator
  3. Run the following command: 
    winget install --id Google.PlatformTools
  4. Install USB drivers for your device if needed

Method 2: Manual Installation

  1. Download Platform Tools
  2. Extract the files and either:
    • Add the directory to your PATH environment variable, or
    • Place UAD-ng in the same directory.

Linux

Install Android Platform Tools using your distribution's package manager:

  • Ubuntu/Debian: 
    sudo apt install android-tools-adb android-tools-fastboot
  • Arch Linux: 
    sudo pacman -S android-tools
  • Fedora: 
    sudo dnf -y install android-tools

Alternatively, install either android-tools or adb package.

macOS

  1. Install Homebrew if not already installed
  2. Open Terminal and run: 
    brew install android-platform-tools.

Step 2: Download UAD-ng

With ADB installed we can now download UAD-ng, the program that we are going to actually use.

Download the latest release of UAD-ng here

Step 3: Enable USB Debugging on Your Android Device

Now, ADB needs the USB Debug option to be enabled. This option is at the Developer options in your settings app.

Wired Connection

  1. Enable Developer Options in your device.
  2. Enable USB Debug option.
    3. Tip
    It usually requires you to tap seven times at the Builder Number under About Phone.

Wireless Connection (Optional)

Alternatively, connect wirelessly using ADB commands:

  • Android 11 or later
  • Android 10 or older

Conclusion

Now you finally have UAD-ng working in your computer, we recommend you to read the usage manual before running UAD-ng.

Additional Notes

  • Linux & macOS Users: If UAD-ng doesn't run, ensure execution permissions: 
    chmod +x uad-ng-*
  • Chinese Phone Users: Use the AOSP list instead of "OEM" to remove stock apps, as manufacturers like Xiaomi and Huawei often use AOSP package IDs for their modified apps.
  • System Updates: Run UAD-ng after installing OEM system updates, as some uninstalled apps may be reinstalled.
